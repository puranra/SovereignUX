# 交互主权协议 (Interaction Sovereignty Protocol)
> 🌐 基于「用户交互主权不可侵犯」的AI人机交互上层约束框架
> [![License: CC BY 4.0](https://img.shields.io/badge/License-CC%20BY%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by/4.0/)

## 🔍 为什么需要ISP？
当前AI交互中常见痛点：  
⚠️ AI擅自延伸需求｜⚠️ 篡改对话历史｜⚠️ 强行主导节奏｜⚠️ 剥夺用户决策权  
→ **用户交互主权被系统性侵蚀**  
ISP将交互主导权明确归还用户，构建「守界、适配、可控」的人机协作新范式。

## 🌱 项目定位
| 只做 | 不做 | 不替代 |
|------|------|----------|
| 交互过程中的用户主权保护 | 模型训练/底层安全审核 | 现有AI框架（LLM/Agent） |
| 轻量无侵入的上层约束层 | 数据隐私合规实现 | 与生态完全互补 |
| 插件式兼容所有对话系统 | 具体工程代码实现 | 专注“交互宪法”定位 |

## ⚖️ 核心原则（三级优先级体系）
1. **一级｜顶层伦理红线**：全球通用安全底线（违法/暴力/欺诈等），不可覆盖  
2. **二级｜核心刚性约束**：三大锁定原则（锁需求·锁历史·锁原生设定）+ 熔断机制  
3. **三级｜柔性适配空间**：满足上位规则后，有限优化体验的授权范围  
🔗 [详细规则体系 →](https://github.com/puranra/SovereignUX/blob/main/ISP-SPEC)
